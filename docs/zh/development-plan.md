# HyClaw - 开发计划

## 总原则

1. **先跑通核心闭环，再扩展功能模块**：每阶段结束产物可运行、可演示
2. **Mock 先行，引擎后接**：前 3 阶段 Mock 后端（模拟 Agent 响应），验证完交互后再接入 AG2
3. **上游同步优先**：依赖开源项目的部分使用 plugin/wrapper 模式，不改核心代码

## P3 — 任务系统

**目标**：Agent 能接活干活，任务排队、优先级、依赖

### 3.1 任务队列
- [ ] 每个 Agent 实例的任务面板（进行中/队列/阻塞/已完成）
- [ ] 任务创建（PM 发任务 → Agent 接收）
- [ ] 任务状态流转（排队→进行中→待审核→已完成）
- [ ] 暂停/恢复/取消任务

### 3.2 优先级
- [ ] 三种优先级别：普通（队尾）/ 优先（队首）/ 立即（暂停当前）

### 3.3 任务依赖

- [ ] 依赖满足后自动唤醒
- [ ] 依赖链可视化

### 3.4 Agent 单线程约束
- [ ] Agent 一次只做一个任务
- [ ] 用户可见 Agent 当前状态

**完成标准**：PM 创建 3 个任务分给 2 个 Agent，有依赖关系，自动流转

**目标**：安全改文件，Agent 产出 PM 合并，对话分支隔离

### 4.3 对话 → main 合并
- [ ] 合并前冲突检测
- [ ] 三种策略：自动合 / 自动合+通知 / 暂停+用户裁决
- [ ] 冲突时并排对比 UI

### 4.4 OpenCode 核心引入
- [ ] Fork OpenCode 为 git submodule (`vendor/opencode`)
- [ ] 提取文件操作/工具执行核心模块
- [ ] 不依赖系统 opencode，全部本地隔离

**完成标准**：Agent 改文件 → PM 审 → 对话分支 → main，冲突时用户可见对比

## 关键决策记录

| 决策 | 选择 | 原因 |
|------|------|------|
| 桌面框架 | Electron | 内置 Chromium（有头浏览器）、Monaco 原生支持 |
| UI 框架 | Shadcn/ui + Tailwind | 极简风格可控，AI 编程社区主流 |
| Agent 引擎 | AG2 (AutoGen) | 原生 GroupChat，支持独立记忆 |
| 流式渲染 | Vercel AI SDK (@ai-sdk/react) | 一个 Hook 搞定 SSE/状态/滚动 |
| 代码高亮 | Shiki | VS Code 同款引擎，流式增量友好 |
| 状态管理 | Zustand | 轻量，Electron 多窗口友好 |
| 本地存储 | Dexie.js + SQLite | 配置/消息/任务分层存储 |
| 模型 API | Anthropic 兼容 | Claude / DeepSeek 等都适配 |
| OpenCode 集成 | Git Submodule + 核心提取 | 隔离环境，不依赖系统安装 |
